After months of building a vertical SaaS with Claude Code, I packaged my workspace setup as a generic template anyone can use.

What it includes:
- `/setup` interview that configures everything from your answers (5 stages)
- Session hooks that auto-save/load state between sessions (never lose context)
- Daily open items tracker that persists across sessions
- Multi-domain routing (separate CLAUDE.md for coding, marketing, financial, support, ops)
- Memory system with 4 types (user, feedback, project, reference)
- Pre-commit safety hooks (catch secrets, brand errors, forbidden files)
- 2 example skills (/daily-standup, /code-review)
- 6 docs (architecture, memory guide, hooks, multi-agent, skills, daily ops)

What it does NOT include:
- No proprietary code from my business
- No specific framework or language dependencies
- Works with any project, any stack

The key differentiator: the `/setup` interview. It asks about your project, tech stack, preferences, and workflow, then generates all config files automatically. A fintech developer and a game studio would get completely different command centers.

MIT Licensed. 31 files. Battle-tested across 30+ agent swarms.
