# Plans

Claude Code creates plan files here during implementation planning.

## Lifecycle
1. **Created** — when you use `EnterPlanMode` or Claude proposes a plan
2. **Active** — while implementation is in progress
3. **Completed** — after implementation is done (can be deleted or archived)

## Format
Plans are markdown files with:
- Context section (why this change)
- Implementation steps
- Files to modify
- Verification steps

Plans are auto-named by Claude. You don't need to manage them manually.
